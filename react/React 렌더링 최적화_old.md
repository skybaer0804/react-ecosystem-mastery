## 1. 렌더링 과정, 리렌더링 조건

### 1.1 렌더링 과정

렌더는 2가지 Phase로 진행됩니다.

1. **Render Phase** :  
    
    컴포넌트를 호출 - React Element 반환
    
    새로운 Virtual DOM 을 생성해주고 재조정(Reconciliation)해서 변경 필요한 부분 체크
    
2. **Commit Phase**
    
    변경 필요한 부분이 있다면 교체.
    

### 1.2 리렌더링 조건

- state 가 바뀌었을 때
- props 가 바뀌었을 때

### 1.3 Profiler devtools

React Developer Tools → Profiler는 리엑트 컴포넌트들이 어떻게 렌더링되는지 보여주는 툴입니다.

## 2. 불필요한 리렌더링을 막자

부모 컴포넌트에서 사용되는 value 들이 변경될때 value를 사용하지 않지만 관련 함수를 사용하는 컴포넌트들도 리렌더링이 되는 구조를 변경합니다.

### **2.1 useCallback → Commit Phase를 막기.**

함수를 **메모이제이션**(기존에 수행한 연산 값을 저장해두고 재사용하는 기법) 해줘도 상관없이 **리렌더링 되지만 효과는 있습니다.**

- **리렌더링이 또 일어나는 이유** : React 동작은 React.createElement로 동작 되기 때문이다.
- **그래도 효과가 있는 이유** : Render Phase는 실행되나 **Commit Phase는 실행되지 않기 때문입니다.**

### 2.2 React.memo → Render Phase 막기.

React.memo를 사용하면 Render Phase를 막을 수 있습니다.

**React.memo는 전달받은 props가 이전 props와 비교했을때 같으면 컴포넌트 리렌더링을 막아주고 마지막으로 렌더링된 결과를 재사용 합니다.**

이때 얕은 비교(원시 타입의 값이 다른지 비교, 참조값이 다른지 비교)를 진행 합니다.

**즉, React.memo(컴포넌트)로 감싸둬 주면 props가 바뀌었을 때 비교할 것이고 바뀌지 않으면 렌더링 되지 않습니다.**

### **2.3 children 잘 사용하기.**

- `React.createElement`는 매번 새로운 요소를 생성합니다. 부모 컴포넌트가 렌더링될 때마다 하위 컴포넌트도 재생성되는 이유 중 하나입니다.
- 하지만 **`children`으로 넘기는 경우, 이미 생성된 React Element를 전달**하는 것이기 때문에 React가 새로운 요소를 생성하지 않고, 불필요한 렌더링을 줄일 수 있습니다.

## 3.  최적화 전에 근본적 코드를 개선

**처음부터 불필요한 렌더링이 발생하지 않도록 작성하는게 중요**합니다.

**미리 최적화 하지 말고, 필요할 때 합시다.**

최적화하는 것도 비용이고 가독성이 떨어져 협업 능률이 떨어집니다.

다 코드이고 내부적 특정 동작을 실행하기에 다 비용이다.

최적화 시도하기 전보다 웹사이트 성능이 안좋아 질 수도 있습니다.
